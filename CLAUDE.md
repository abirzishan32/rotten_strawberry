## What this is

A Letterboxd-inspired movie review/diary app built on Expo Router, styled with NativeWind (Tailwind for React Native). Movie data is live from TMDB; user data (auth, profiles, reviews/diary, favorites, watchlist) is backed by **Supabase** (Postgres + Auth + RLS). Only `theme` preference and `recent searches` persist locally (Zustand + AsyncStorage). Browsing is open to guests; personal actions prompt sign-in.

## Environment / secrets

TMDB auth uses a **v4 Read Access Token** (bearer), stored in `.env` as `TMDB_API_KEY` (gitignored). `app.config.ts` reads `process.env.TMDB_API_KEY` at config-eval time (Expo CLI auto-loads `.env` into `process.env`, no `EXPO_PUBLIC_` prefix needed since it's read server-side) and injects it into `extra.tmdbApiKey`. `services/tmdb/client.ts` pulls it back out via `Constants.expoConfig.extra.tmdbApiKey` and sends it as `Authorization: Bearer <token>` â€” this keeps the raw key out of app.json/JS source, though it's still embedded in the compiled app config (acceptable for a client-only TMDB read token).

Supabase uses the same `.env` â†’ `app.config.ts` `extra` â†’ `Constants.expoConfig.extra` pattern: `SUPABASE_URL` and `SUPABASE_ANON_KEY` (anon key only â€” public-safe because RLS guards the data). **Never add the `service_role` key** to `.env` or the app; it bypasses RLS. After editing `.env`, restart the dev server so `app.config.ts` re-reads it.

## Architecture

- **Routing (`app/`)**: `app/_layout.tsx` is the root `Stack` wrapping the app in `GestureHandlerRootView` â†’ `SafeAreaProvider` â†’ `QueryClientProvider` (client from `services/query-client.ts`) â†’ navigation `ThemeProvider` â†’ `ErrorBoundary`. `(tabs)` is the anchor group with 4 tabs (Home, Search, Activity, Profile) defined in `app/(tabs)/_layout.tsx`. `movie/[id]`, `browse/[category]`, `browse/genre/[id]` are pushed outside the tabs group (so the tab bar hides automatically), and `add-log` / `settings` / `auth` are `presentation: 'modal'` screens registered directly in the root `Stack`.
- **Bottom sheets use `BottomSheet`, not `BottomSheetModal`**: `components/common/bottom-sheet.tsx` wraps `@gorhom/bottom-sheet`'s plain, fully-controlled `BottomSheet` (open/closed via an `index` prop) rather than `BottomSheetModal`'s portal + imperative present/dismiss stack, which was dropped after it got stuck on a close/reopen cycle. There is no `BottomSheetModalProvider` in the tree â€” don't add one or reach for `BottomSheetModal` for new sheets.
- **Logging a movie**: there is no "Add Log" tab. The `add-log` modal (`app/add-log.tsx`) is reached from the "Log" FAB on `movie/[id]` (search â†’ movie â†’ log). It's registered as a modal in the root `Stack`.
- **Cinematic Atlas (`app/map.tsx` + `components/map/`)**: a 3D-globe screen (pushed route `/map`, reached from the profile header globe icon / banner) showing which countries the user has watched films from. `hooks/use-map-data.ts` derives everything: watched movies don't store country, so it fetches TMDB `movieDetails` for each unique watched movie (via `useQueries`, reusing the `movieDetails` cache) and aggregates by `production_countries[0].iso_3166_1`. `constants/country-geo.ts` maps ISO-2 â†’ name + continent (+ `isoToFlag` via regional-indicator emoji; the lat/lon fields are vestigial from the old dots map and unused by the globe).
  - **The globe is a real choropleth 3D globe** (`interactive-world-globe.tsx`): a `react-native-webview` hosting **globe.gl (Three.js)** with **Natural Earth country boundaries** (GeoJSON via jsDelivr) â€” countries are extruded + coloured by watch count, drag-rotate / pinch-zoom, tap posts the ISO back over `postMessage`. globe.gl + the boundaries load from CDNs at runtime (nothing bundled). RN passes counts in via `injectedJavaScriptBeforeContentLoaded` and refreshes them with `injectJavaScript(window.__update__(...))` as detail queries stream in. Country match is on `ISO_A2_EH || ISO_A2`; a few countries without their own admin-0 polygon (e.g. Hong Kong) won't highlight. Requires `react-native-webview` (in Expo Go already; a custom dev client needs a rebuild).
  - Country taps open `country-detail-sheet.tsx` (the shared `AppBottomSheet`, controlled by nulling the selected iso). The screen is intentionally always-dark/cinematic (hardcoded, not theme-aware). There is **no** achievement/badge system, and non-flag decorative emojis are avoided (Ionicons instead).
- **TMDB layer**: `services/tmdb/client.ts` (axios instance) â†’ `services/tmdb/movies.ts` (one function per endpoint: lists, details, credits, videos, similar, recommendations, search, discover, genres) â†’ `hooks/queries/*` (TanStack Query wrappers; `use-movie-list.ts` deliberately has *two* query keys â€” `movieList` for single-page home carousels and `movieListInfinite` for paginated browse grids â€” sharing one key would corrupt the cache since `useQuery` and `useInfiniteQuery` store different shapes). `constants/query-keys.ts` is the single source of truth for query keys.
- **Supabase backend (`services/supabase/`)**: auth + all user-generated data (profiles, reviews/diary, favorites, watchlist, favorite_films) live in Supabase. `client.ts` builds one `createClient<Database>` with the AsyncStorage session adapter + `detectSessionInUrl: false` and an `AppState` auto-refresh listener; per-domain modules (`auth`, `profiles`, `reviews`, `favorites`, `watchlist`, `favorite-films`) wrap the queries and map snake_case rows â†” the app's camelCase types via `types.ts`. `hooks/queries/use-{reviews,favorites,watchlist,favorite-films,profile}.ts` are the TanStack Query wrappers (favorites/watchlist/favorite-film toggles + reorder are optimistic). The SQL schema + RLS + signup trigger live in `supabase/schema.sql` (run once in the dashboard, and **re-run after pulling schema changes** â€” it's `add column/create table if not exists`, so it migrates in place; see `supabase/README.md`). **Only the anon key is bundled** (via `app.config.ts` `extra`, same pattern as TMDB) â€” RLS is the real protection; the `service_role` key must never enter the app. Row types in `types.ts` are `type` aliases, not `interface`s, and each table carries `Relationships: []`, because postgrest-js's `GenericSchema` constraint otherwise collapses the whole `Database` to `never`.
  - **Reviews carry four rating metrics**: `rating` (the overall "Movie" score) plus `rating_ending` / `rating_rewatchability` / `rating_pacing`, each 0.5â€“5 (0 = unset). `types/review.ts` `REVIEW_METRICS` is the single source of truth for the metric keyâ†’label list; the add-log form and `components/review/review-metrics.tsx` both map over it. `reviews.user_id` FKs to `profiles(id)` (for author embedding), so `reviews.createReview` calls `profilesApi.ensureProfile(userId)` first to guarantee the row exists.
  - **`favorite_films`** is the curated max-4 profile showcase (ordered by `position`), separate from the unlimited hearted `favorites`. `components/profile/favorite-films-row.tsx` is a hand-rolled Gesture Handler + Reanimated drag-to-reorder row (no extra dep). The profile "Diary" tab renders `components/profile/diary-list.tsx` (month-grouped calendar view). Profile bio (â‰¤200 words) + location are edited in the `edit-profile` modal; favourite films are added via the `pick-favorite-film` modal.
  - **Auth is session state, not React-owned**: `store/auth-store.ts` (Zustand) holds the session; `hooks/use-auth-init.ts` (called once in the root layout) is the *only* thing that calls `setSession`, driven by `supabase.auth.onAuthStateChange`. Browsing is guest-allowed; `hooks/use-require-auth.ts` gates write actions by pushing the `/auth` modal. **Never subscribe a navigator layout (e.g. `(tabs)/_layout.tsx`) to the auth store** â€” read it imperatively with `useAuthStore.getState()` in handlers. A subscription there re-renders the tab tree on every auth event and can spin React Navigation's per-screen option effects into a "Maximum update depth exceeded" loop.
- **Local-only persistence (`store/`)**: `theme-store` (preference, synced into NativeWind's `colorScheme.set`) and `search-store` (recent searches) remain Zustand + AsyncStorage â€” pure device UX, never synced. (Diary/favorites/watchlist used to be local Zustand stores; they now live in Supabase.)
- **Theming**: NativeWind (`darkMode: 'class'`) drives `className`-based styling everywhere. `hooks/use-app-theme.ts` is the bridge for the few places that can't use `className` (icon `color` props, the navigation `ThemeProvider`, status bar style, gradients, blur tint) â€” it resolves NativeWind's `useColorScheme()` against the hex token table in `constants/theme.ts` (`Colors.light` / `Colors.dark`, kept in sync with `tailwind.config.js`). To add a new semantic color, add it in both places.
- **Component layers**: `components/common/` is app-agnostic UI (Button, Input, Chip, Skeleton, Modal, BottomSheet wrapper, ThemeToggle, ErrorBoundary â€” barrel in `index.ts`). `components/movie/` and `components/review/` are domain components (MovieCard, MovieCarousel, MovieGridScreen for paginated grids, SearchFilterSheet, StarRating, ReviewCard). `MovieGridScreen` is shared by both `browse/[category]` and `browse/genre/[id]` to avoid duplicating the grid+pagination+header chrome.
- **Path aliasing**: `@/*` maps to the project root (`tsconfig.json`).
- **New Architecture**: `newArchEnabled: true` â€” runs on Fabric/TurboModules. Reanimated 4 uses the split `react-native-worklets` package; the Babel plugin is `react-native-worklets/plugin`, not the old `react-native-reanimated/plugin`.
- **`babel-preset-expo` is an explicit top-level devDependency** â€” without it, npm's dedupe nests it under `node_modules/expo/node_modules/`, which our custom `babel.config.js` (needed for NativeWind + worklets) can't resolve from the project root.
