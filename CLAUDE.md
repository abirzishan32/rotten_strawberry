## What this is

A Letterboxd-inspired movie review/diary app built on Expo Router, styled with NativeWind (Tailwind for React Native), backed by the TMDB API. No custom backend — auth, followers, and social data are static/local placeholders; movie data is live from TMDB and user logs/reviews/favorites/theme preference persist locally via Zustand + AsyncStorage.

## Commands

- `npm start` — start the Expo dev server (Metro). Press `i`/`a`/`w` in the terminal to open iOS/Android/web.
- `npm run ios` / `npm run android` / `npm run web` — start the dev server targeting a specific platform directly.
- `npm run lint` — run ESLint via `expo lint` (flat config in `eslint.config.js`, extends `eslint-config-expo`).
- `npx tsc --noEmit` — typecheck. Route param types depend on `.expo/types/router.d.ts`, which is regenerated by `expo start`/`expo export` whenever `app/` changes — run the dev server at least once after adding/removing routes before trusting typed-route errors.
- `npm run reset-project` — leftover from the original template; do not run, it would wipe this app's `app/` directory.
- No test runner is configured in this project.

## Environment / secrets

TMDB auth uses a **v4 Read Access Token** (bearer), stored in `.env` as `TMDB_API_KEY` (gitignored). `app.config.ts` reads `process.env.TMDB_API_KEY` at config-eval time (Expo CLI auto-loads `.env` into `process.env`, no `EXPO_PUBLIC_` prefix needed since it's read server-side) and injects it into `extra.tmdbApiKey`. `services/tmdb/client.ts` pulls it back out via `Constants.expoConfig.extra.tmdbApiKey` and sends it as `Authorization: Bearer <token>` — this keeps the raw key out of app.json/JS source, though it's still embedded in the compiled app config (acceptable for a client-only TMDB read token).

## Architecture

- **Routing (`app/`)**: `app/_layout.tsx` is the root `Stack` wrapping the app in `GestureHandlerRootView` → `SafeAreaProvider` → `QueryClientProvider` → `BottomSheetModalProvider` → navigation `ThemeProvider`. `(tabs)` is the anchor group with 5 tabs (Home, Search, Add Log, Activity, Profile) defined in `app/(tabs)/_layout.tsx`. `movie/[id]`, `browse/[category]`, `browse/genre/[id]` are pushed outside the tabs group (so the tab bar hides automatically), and `add-log` / `settings` are `presentation: 'modal'` screens registered directly in the root `Stack`.
- **The "Add Log" tab is not a real screen**: its `tabPress` listener in `(tabs)/_layout.tsx` calls `e.preventDefault()` and pushes `/add-log` (the modal) instead. `app/(tabs)/add-log.tsx` only exists as a `<Redirect>` fallback for deep links — the actual UI lives in `app/add-log.tsx`.
- **TMDB layer**: `services/tmdb/client.ts` (axios instance) → `services/tmdb/movies.ts` (one function per endpoint: lists, details, credits, videos, similar, recommendations, search, discover, genres) → `hooks/queries/*` (TanStack Query wrappers; `use-movie-list.ts` deliberately has *two* query keys — `movieList` for single-page home carousels and `movieListInfinite` for paginated browse grids — sharing one key would corrupt the cache since `useQuery` and `useInfiniteQuery` store different shapes). `constants/query-keys.ts` is the single source of truth for query keys.
- **Local persistence (`store/`)**: four independent Zustand stores, each persisted to AsyncStorage — `theme-store` (preference, synced into NativeWind's `colorScheme.set`), `log-store` (diary/review entries), `favorites-store`, `search-store` (recent searches). There is no server sync; this *is* the data layer for anything user-generated.
- **Theming**: NativeWind (`darkMode: 'class'`) drives `className`-based styling everywhere. `hooks/use-app-theme.ts` is the bridge for the few places that can't use `className` (icon `color` props, the navigation `ThemeProvider`, status bar style, gradients, blur tint) — it resolves NativeWind's `useColorScheme()` against the hex token table in `constants/theme.ts` (`Colors.light` / `Colors.dark`, kept in sync with `tailwind.config.js`). To add a new semantic color, add it in both places.
- **Component layers**: `components/common/` is app-agnostic UI (Button, Input, Chip, Skeleton, Modal, BottomSheet wrapper, ThemeToggle, ErrorBoundary — barrel in `index.ts`). `components/movie/` and `components/review/` are domain components (MovieCard, MovieCarousel, MovieGridScreen for paginated grids, SearchFilterSheet, StarRating, ReviewCard). `MovieGridScreen` is shared by both `browse/[category]` and `browse/genre/[id]` to avoid duplicating the grid+pagination+header chrome.
- **Path aliasing**: `@/*` maps to the project root (`tsconfig.json`).
- **New Architecture**: `newArchEnabled: true` — runs on Fabric/TurboModules. Reanimated 4 uses the split `react-native-worklets` package; the Babel plugin is `react-native-worklets/plugin`, not the old `react-native-reanimated/plugin`.
- **`babel-preset-expo` is an explicit top-level devDependency** — without it, npm's dedupe nests it under `node_modules/expo/node_modules/`, which our custom `babel.config.js` (needed for NativeWind + worklets) can't resolve from the project root.
